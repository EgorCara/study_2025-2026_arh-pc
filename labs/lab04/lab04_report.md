3. Транслятор NASM
   1. NASM превращает текст программы в объектный код. Например, для компиляции приведённого выше текста 
программы «Hello World» необходимо написать: nasm -f elf hello.asm. С помощью команды ls проверим, что объектный файл был создан.
![Рис.6](/home/eakara/Pictures/Screenshots/Screenshot from 2025-11-20 17-54-04.png)
Рис.6

4. Расширенный синтаксис командной строки NASM
   1. Выполним следующую команду: nasm -o obj.o -f elf -g -l list.lst hello.asm
Данная команда скомпилирует исходный файл hello.asm в obj.o (опция -o позволяет
задать имя объектного файла, в данном случае obj.o), при этом формат выходного файла
будет elf, и в него будут включены символы для отладки (опция -g), кроме того, будет создан
файл листинга list.lst (опция -l).
С помощью команды ls проверили, что файлы были созданы.
![Рис.7](/home/eakara/Pictures/Screenshots/Screenshot from 2025-11-20 17-57-50.png) 
Рис.7

5. Компоновщик LD
   1. Как видно из схемы на рис. 4.3, чтобы получить исполняемую программу, объектный файл
необходимо передать на обработку компоновщику:
ld -m elf_i386 hello.o -o hello
![Рис.8](/home/eakara/Pictures/Screenshots/Screenshot from 2025-11-20 20-38-25.png)
Рис.8
С помощью команды ls проверим, что исполняемый файл hello был создан.
![Рис.9](/home/eakara/Pictures/Screenshots/Screenshot from 2025-11-20 20-39-08.png)
Рис.9
Компоновщик ld не предполагает по умолчанию расширений для файлов, но принято
использовать следующие расширения:
• o – для объектных файлов;
• без расширения – для исполняемых файлов;
• map – для файлов схемы программы;
• lib – для библиотек.
Ключ -o с последующим значением задаёт в данном случае имя создаваемого исполняемого файла.
Выполним следующую команду:
ld -m elf_i386 obj.o -o main
